## 一、项目概述

**项目名称**：配煤计算器  
**目标用户**：煤炭贸易公司配煤人员（欣欣姐姐等）  
**核心用途**：手动输入多种煤的指标（灰分、硫分、挥发分、粘结）及配比（单位：成），按**算术平均法**（除以煤种数量）计算混合煤的综合指标，并与可自由设定的钢厂目标范围比对，判断是否达标。

**技术形态**：单文件 HTML/CSS/JS 网页应用，适配手机与电脑，无后端依赖。

## 二、需求确认记录（原始需求 + 后期调整）

### 2.1 原始需求（v1.0 之前）

1. **核心指标**：灰分(A%)、硫分(S%)、挥发分(V%)、粘结(G)。  
2. **配比单位**：使用“成”（1成 = 10%），计算时乘以 0.1 作为权重。  
3. **配比方式**：手动输入每行煤种的份数（成），总和任意。  
4. **煤种库**：默认提供 8 种煤，支持动态添加新煤种（手动填写所有参数）和删除煤种（至少保留一种）。  
5. **目标范围**：四个指标均可独立设置下限/上限，用户自由修改。  
6. **计算模式**：用户修改数据后**不自动计算**，需点击对应确认按钮才触发计算与达标判定。  
7. **空值检查**：点击确认时，检查所有必填字段，若缺失则弹出友好提示，用户可选择返回填写或强制计算（缺失项自动视为 0）。  
8. **界面布局**：三个独立卡片模块——  
   - 混合煤实际指标（显示计算结果及达标状态）  
   - 煤种指标 & 配比（可编辑表格）  
   - 钢厂目标范围（四个指标的上下限）  
9. **视觉风格**：标题左侧竖条 + 图标 + 文字，表格内文字及三个按钮字号较大，移动端适配。  
10. **按钮与交互**：  
    - 添加新煤种（绿色）→ 依次弹窗输入所有参数。  
    - 确认指标和配比（蓝色）→ 检查表格完整性 → 同步数据 → 计算并刷新结果 → 成功提示。  
    - 确认目标范围（蓝色）→ 检查目标范围完整性 → 处理上下限颠倒（弹出确认对话框） → 更新达标状态 → 成功提示。  
    - 删除按钮（🗑️）→ 删除后自动计算。  
11. **网页标题**：固定为“配煤计算器”。  
12. **提示语优化**：缺失时显示具体煤种+字段名称（如“神华9层的煤价”），目标范围显示“灰分的下限”等中文。

### 2.2 后期调整（v1.11 → v1.14）

| 调整项 | 说明 |
|--------|------|
| **算法变更** | 从行业标准的**加权平均法**（除以总权重）改为**算术平均法**（除以煤种数量）。用户亲戚确认：混合指标 = Σ(指标 × 配比×0.1) / 煤种数量。 |
| **空值强制计算** | 用户选择“继续计算”后，缺失的数值自动当作 0，不再产生 NaN。 |
| **目标范围上下限颠倒** | 点击“确认目标范围”时，若发现下限 > 上限，自动交换并弹窗显示交换后的数值，用户确认后才应用；取消则不更新。 |
| **成功提示回归** | 修复 v1.13 中“确认目标范围”后不弹出成功提示的问题，v1.14 恢复正常。 |
| **底部说明更新** | 明确写出计算规则：“混合指标计算方式：按配比(成)*0.1加权求和后，除以煤种数量”。 |

## 三、功能规格说明（v2.0）

### 3.1 数据模型

**煤种对象**（JavaScript 对象结构）：

```javascript
{
  name: string,      // 煤种名称
  price: number,     // 煤价（元/吨）
  ash: number,       // 灰分%
  sulfur: number,    // 硫分%
  volatile: number,  // 挥发分%
  glue: number,      // 粘结G
  ratio: number      // 配比（成）
}
