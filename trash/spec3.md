## 一、项目概述

**项目名称**：配煤计算器  
**目标用户**：煤炭贸易公司配煤人员  
**核心用途**：手动输入多种煤的指标（灰分、硫分、挥发分、粘结）及配比（单位：成），按**算术平均法**（除以煤种数量）计算混合煤的综合指标，并与可自由设定的钢厂目标范围比对，判断是否达标。

**技术形态**：单文件 HTML/CSS/JS 网页应用，适配手机与电脑，无需后端，数据本地处理。

## 二、需求确认记录（原始需求）

1. **核心指标**：灰分(A%)、硫分(S%)、挥发分(V%)、粘结(G)。  
2. **配比单位**：使用“成”（1成 = 10%），计算时需乘以 0.1作为权重。  
3. **配比方式**：手动输入每行煤种的份数（成），总和任意。  
4. **煤种库**：默认提供 8 种煤，支持动态添加新煤种（手动填写所有参数）和删除煤种（至少保留一种）。  
5. **目标范围**：四个指标均可独立设置下限/上限，用户自由修改。  
6. **计算模式**：用户修改数据后**不自动计算**，需点击对应区域的确认按钮才触发计算与达标判定。  
7. **空值检查**：点击确认时，检查所有必填字段，若缺失则弹出友好提示，用户可选择返回填写或强制计算（缺失项自动视为 0）。  
8. **界面布局**：三个独立卡片模块——  
   - 混合煤实际指标（显示计算结果及达标状态）  
   - 煤种指标 & 配比（可编辑表格）  
   - 钢厂目标范围（四个指标的上下限）  
9. **视觉风格**：标题左侧竖条 + 图标 + 文字，表格内文字及三个按钮字号较大，移动端适配。  
10. **按钮与交互**：  
    - 添加新煤种（绿色）→ 依次弹窗输入所有参数 → 添加到表格末尾 → 提示“已添加，请点击确认按钮更新计算结果” → 不自动计算。  
    - 确认指标和配比（蓝色）→ 检查表格完整性 → 同步数据 → 计算并刷新结果 → 成功提示。  
    - 确认目标范围（蓝色）→ 检查目标范围完整性 → 处理上下限颠倒（弹出确认对话框） → 更新达标状态 → 成功提示。  
    - 删除按钮（🗑️）→ 删除煤种后提示“已删除，请点击确认按钮更新计算结果” → 不自动计算。
    - 所有计算（混合指标、达标状态）仅在点击“确认指标和配比”或“确认目标范围”时触发。  
11. **网页标题**：固定为“配煤计算器”。  
12. **提示语优化**：缺失时显示具体煤种+字段名称（如“神华9层的煤价”），目标范围显示“灰分的下限”等中文。

## 三、功能规格说明

### 3.1 数据模型

**煤种对象**（JavaScript 对象结构）：

    {
      name: string,      // 煤种名称
      price: number,     // 煤价（元/吨）
      ash: number,       // 灰分%
      sulfur: number,    // 硫分%
      volatile: number,  // 挥发分%
      glue: number,      // 粘结G
      ratio: number      // 配比（成）
    }

**目标范围对象**：

    targetBounds = {
      ash:   { min: number, max: number },
      sulfur:{ min: number, max: number },
      volatile:{ min: number, max: number },
      glue:  { min: number, max: number }
    }

### 3.2 加权计算算法

- 单个煤种权重 = 配比（成） × 0.1
- 加权和 = Σ( 煤种指标 × 权重 )
- 混合指标 = 加权和 / 煤种数量 （注意：分母是煤种数量，而不是总权重）

> 示例：2 种煤，A 配比 2 成，灰分 10；B 配比 3 成，灰分 20。  
> 权重：A=0.2，B=0.3  
> 加权和 = 10×0.2 + 20×0.3 = 2 + 6 = 8  
> 混合灰分 = 8 / 2 = 4.0

### 3.3 达标判定

对于每个指标（除煤价外），若 min ≤ 实际值 ≤ max 则显示“达标”（绿色标签），否则“超标”（红色标签）。煤价仅显示数值及“成本指标”灰色标签。

### 3.4 空值检查逻辑

- **表格缺失检查**：遍历每一行，检查煤种名称、煤价、各指标、配比是否为有效数字或非空。缺失时收集 `<煤种名>的<字段名>` 字符串。
- **目标范围缺失检查**：检查 8 个输入框，缺失时收集“灰分的下限”等中文名称。
- **用户交互**：若存在缺失，弹窗列出所有缺失项，并询问“是否仍继续计算？（未填数值将自动视为0）”。用户选“是”则继续，所有空值或无效值在同步时被设为 0；选“否”则中止。

### 3.5 目标范围上下限颠倒处理

- 点击“确认目标范围”时，对每个指标的 min/max 进行检查。
- 若 min > max，则自动交换数值，并弹窗提示：“以下目标范围下限大于上限，已自动交换为：灰分: 5.00 ~ 15.00 … 是否确认应用此范围？”
- 用户点击“确认”则应用交换后的值并更新达标状态；点击“取消”则不做任何改变，计算保持原样。

### 3.6 用户交互流程

1. 用户修改任意表格数值或目标范围输入框 → 不会自动计算。  
2. 点击“确认指标和配比”：
   - 检查表格空值 → 若有缺失且用户选“继续” → 同步表格数据（空值→0） → 重新计算混合指标 → 刷新界面 → 弹窗成功提示。
3. 点击“确认目标范围”：
   - 检查目标范围空值 → 若有缺失且用户选“继续” → 检查上下限颠倒 → 若有颠倒且用户确认交换 → 更新目标范围 → 重新计算达标状态 → 弹窗成功提示。
4. 添加新煤种：顺序弹出 7 个输入框，验证后添加到表格末尾，刷新表格显示，弹出提示“请点击确认按钮更新计算结果”，不自动计算。
5. 删除煤种：直接删除，刷新表格显示，弹出提示“请点击确认按钮更新计算结果”，不自动计算。

## 四、界面结构（最终版）
