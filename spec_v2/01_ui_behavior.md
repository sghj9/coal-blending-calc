# 01 — UI 交互规则

> 关联模块：`js/ui.js`、`js/validation.js`
> 原始内容来源：`spec/frontend_spec.md` §3.3、§3.4.4–3.4.7

## 一、达标判定显示

对于每个指标（除煤价外），若 min ≤ 实际值 ≤ max 则显示"达标"（绿色标签），否则"超标"（红色标签）。煤价仅显示数值及"成本指标"灰色标签。

## 二、配比优化 — 输入

| 输入 | 来源 | 说明 |
|------|------|------|
| 煤种列表 | 当前表格中的煤种（不含配比列） | 取 name / price / ash / sulfur / volatile / glue |
| 目标范围 | 当前"钢厂目标范围"输入框的值 | 四个指标的 min / max |

## 三、配比优化 — 输出

点击按钮后，在"钢厂目标范围"卡片下方展示优化结果卡片：

| 字段 | 说明 |
|------|------|
| 建议配比列表 | 每种煤的建议配比（成），MILP 天然输出 1 位小数，与应用后结果一致 |
| 总配比 | Σ 建议配比（成） |
| 综合煤价 | 最优目标函数值（¥/t） |
| 预期混合指标 | 灰分/挥发分显示达标状态，硫分/粘结仅作参考值显示 |
| 状态提示 | 成功（绿色）或失败原因（红色） |

## 四、配比优化 — 交互

- "🔍 优化配比建议"按钮位于"钢厂目标范围"卡片内
- 点击后触发计算，结果展示在下方的动态卡片中
- "应用建议"按钮：将建议配比填入煤种表格的各行配比输入框
- 应用后不自动计算，用户仍需点击"确认指标和配比"

## 五、配比优化 — 边界与错误处理

| 情况 | 行为 |
|------|------|
| 煤种数量 = 0 | 提示"请先添加煤种" |
| 目标范围未填写 | 使用默认值（同 fetchTargetBoundsFromInputs 逻辑） |
| 无可行解 | 显示"当前煤种无法在目标范围内配出合格混合煤，请放宽目标范围或调整煤种" |
| 优化后总配比 < 10 成 | 正常（不足十成部分不贡献指标） |
| 所有煤价相同 | 找任意可行解（成本等价） |
| 目标下限 = 上限 | 作为等式约束处理（算法中为两个 ≤ 约束夹逼） |
| MILP 无可行解但 LP 有解 | 极为罕见（LP 连续解恰好落在整数网格外且舍入后越界）；此时提示放宽目标范围 |
| 求解超时（n > 20 煤种） | 分支树过大时，返回"优化超时，请减少煤种数量或放宽约束" |
