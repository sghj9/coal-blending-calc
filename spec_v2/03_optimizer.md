# 03 — 优化算法（独立）

> 关联模块：`js/optimizer.js`
> 原始内容来源：`spec/frontend_spec.md` §3.4.1–3.4.3

## 一、功能概述

用户设定目标范围后，点击"🔍 优化配比建议"按钮，系统自动求解**满足灰分、挥发分和硫分约束且综合煤价最低**的配比组合（粘结不作为优化约束，仅作为参考值展示）。

## 二、数学模型（MILP）

将配比优化建模为**混合整数线性规划（MILP）**问题。配比取值域限定为 0.01 成的整数倍，求解结果天然精确到 2 位小数，无需舍入。

混合指标采用标准加权平均：`Σ(指标_i × r_i) / Σr_i`。因优化器强制总配比 `Σr_i = 10`（等式约束），分母为常数 10，约束保持线性。

**变量变换**：令 y_i = 100 × r_i，y_i ∈ {0, 1, …, 1000} 为整数变量。
将 r_i = y_i / 100、Σr_i = 10（即 Σy_i = 1000）代入加权平均：

- **决策变量**：y_i（整数），共 n 个。每 1 单位 y_i 代表 0.01 成配比
- **目标函数**：min Z = Σ(p_i × y_i × 0.001) —— 等价为最小化综合煤价
- **约束条件**（以 y_i 表示）：
  1. 灰分：A_min ≤ Σ(a_i × y_i × 0.001) ≤ A_max
  2. 挥发分：V_min ≤ Σ(v_i × y_i × 0.001) ≤ V_max
  3. 硫分：S_min ≤ Σ(s_i × y_i × 0.001) ≤ S_max
  4. 总配比：Σ y_i = 1000（即 Σ r_i = 10，等式约束）
  5. y_i ∈ Z⁺（0 ≤ y_i ≤ 1000 的整数）

> 粘结因洗煤过程中化学变化不可预判，不作为优化约束，仅在结果中作为参考值展示。

求解后 r_i = y_i / 100 即配比（成），天然 2 位小数，**无舍入偏差**。混合指标按 `Σ(指标_i × r_i) / Σr_i` 计算后四舍五入到 2 位小数。

> 约束系数理解：每 1 单位 y_i（= 0.01 成配比）贡献 指标 × 0.001 到加权指标（分母固定为 10 成）。
> 连续 LP 四舍五入前后不一致的问题不复存在——变量域本身就在网格上。

> 未来扩展：若引入「特殊煤总配比=8」约束，需将等式 RHS 改为 800、系数由 0.001 改为 0.00125（=1/800）。本次不实现。

## 三、算法：Branch-and-Bound + 两阶段单纯形法

采用 **Branch-and-Bound（分支定界）** 框架，以两阶段单纯形法求解每个节点的 LP 松弛：

1. **LP 松弛**：忽略 y_i 的整数约束，用单纯形法求解连续 LP。若该节点 LP 无可行解或下界 ≥ 已知最优整数解 → **剪枝**
2. **分支**：选一个非整数的 y_k，生成两个子节点 —— y_k ≤ ⌊y_k⌋ 和 y_k ≥ ⌈y_k⌉ —— 通过动态添加约束行实现
3. **搜索策略**：深度优先 + 最优下界优先（先探索可能更优的分支，加速找到好解以便剪枝）
4. **终止**：所有节点被剪枝或遍历完毕 → 返回已找到的最优整数解

- 单纯形法：两阶段（Phase I 找初始可行基，Phase II 求最优解），Bland's rule 防退化
- 分支约束通过临时追加行到 tableau 实现，回溯时恢复
- 规模：n 种煤 → 最多 100^n 个整数点，但 branch-and-bound 实际探索节点极少（紧约束剪枝有效）
- 实现文件：`js/optimizer.js`
