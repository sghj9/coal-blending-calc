# spec_v2 — 配煤计算器规格文档

> 原始规格文档 `spec/frontend_spec.md` 保留不删、不修改。本目录为拆分后的模块化规格。

## 目录结构

| 文件 | 内容 | 角色 |
|------|------|------|
| `00_core.md` | 产品核心行为（唯一真相） | 项目结构、数据模型、端点/接口、边界、明确不做 |
| `01_ui_behavior.md` | UI 交互规则 | 达标判定显示、优化功能的输入/输出/交互/错误处理 |
| `02_calculation.md` | 计算逻辑（公式级） | 加权计算算法、达标判定逻辑 |
| `03_optimizer.md` | 优化算法（独立） | MILP 数学模型、Branch-and-Bound + 单纯形法 |
| `90_experiments.md` | 实验性设计 | 待办与未来扩展方向（roadmap） |
| `99_archived.md` | 废弃内容 | 已移除或推翻的设计记录 |
| `README.md` | 本文档 | 目录索引与使用说明 |

## 阅读顺序

- **新人上手**：先读 `00_core.md` 了解产品全貌，再按需深入各模块
- **开发实现**：以 `00_core.md` 为唯一真相源，结合具体模块 spec 指导编码
- **测试编写**：对照 `00_core.md` 的边界条件 + 模块 spec 的行为定义

## 更新原则

- `00_core.md` 仅在新功能语义、行为定义变更、关键约束或边界条件变化时更新
- 模块 spec（01–03）随对应模块行为变更同步更新
- `90_experiments.md` 随时可追加新想法
- `99_archived.md` 仅在功能废弃时追加
